Update generated OpenSpec configuration for OpenCode and Pi only. Run the generators, copy their outputs into the repo, and verify the copies. Do not use this skill for normal OpenSpec workflows.

## Workflow

1. Install the latest OpenSpec CLI:
   ```bash
   npm install -g @fission-ai/openspec@latest
   ```
2. Create separate temporary staging directories outside the repository for OpenCode and Pi.
3. Run `openspec init --tools opencode` in the OpenCode staging directory and `openspec init --tools pi` in the Pi staging directory.
4. Copy the generated OpenCode command files from the OpenCode staging `.opencode/commands/` directory into `dotfiles/opencode/commands/`.
5. Copy the generated Pi prompt files whose names start with `opsx-` from the Pi staging `.pi/prompts/` directory into `dotfiles/pi/prompts/`.
6. Copy the generated OpenSpec skill directories whose names start with `openspec-` into `dotfiles/agents/skills/`.
   - Never delete or modify `update-openspec`; it is a local/manual skill and is not generated by OpenSpec.
   - Remove stale generated `openspec-*` directories from the target before copying.
   - If OpenCode and Pi generate the same `openspec-*` skill files, verify they match before replacing them.
   - If they differ, stop and report the mismatch instead of choosing one silently.
7. Verify the copy by comparing only the generated source and target subsets and counting files or directories.
   - Expected baseline is usually 4 OpenCode commands, 4 Pi prompts, and 4 generated skills; report deviations.
8. Grep the copied generated files for notable upstream tool references and report any findings.
9. Delete only the temporary staging directories when finished.

## Rules

- Always update both OpenCode and Pi in one run.
- Do not use the repository's `dotfiles/opencode/`, `dotfiles/pi/`, or `.opencode/` directories as staging locations.
- Do not manually rewrite generated content.
- Do not delete, overwrite, or otherwise modify `update-openspec`.
- Do not use this skill for regular OpenSpec proposal, exploration, apply, or archive workflows.
- Do not delete any repository directories; only remove temporary staging directories.
- Keep the copy limited to generated OpenSpec outputs.

## Output

Return a concise summary with:

- update result for OpenCode and Pi
- copied command, prompt, and skill counts
- verification results, including any count deviations
- any tool-reference findings
- cleanup result
